! WHY THIS LIST EXISTS вЂ” the dual-layer strategy
!
! Prime Video delivers ads down two different "roads", chosen server-side per
! session (device, title, account, region, A/B bucket):
!
!   вЂў Java road  вЂ” ad break signalled to the client as a media3 AdPlaybackState
!                  ad group. Handled by the bytecode PATCH (skipAds*).
!   вЂў Native road вЂ” ad stitched server-side (SGAI) and delivered through Amazon's
!                  native pipeline + native libcurl. The bytecode patch CANNOT
!                  see this; it runs below the Java HTTP layer. THIS LIST is the
!                  tool that reaches it, because DNS blocking operates below HTTP.
!
! Neither layer alone is complete. Run the patch AND this list together.
!
! в”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђв”Ђ
